react-native iOS 初始化

iOS 部分

object-c 基础语法

object-c 是 c 的一个超集，是在 c 上盖了一层面向对象的特性，支持所有 c 的代码直接运行

object-c 对象之间调用方法不是直接调用，而是类似一种消息传递的方式去调用的，所以方法调用不会在编译的时候检测。

调用方法视为对对象发送消息，所有方法都被视为对消息的回应。所有消息处理直到运行时（runtime）才会动态决定，并交由类别自行决定如何处理收到的消息。也就是说，一个类别不保证一定会回应收到的消息，如果类别收到了一个无法处理的消息，程序只会抛出异常，不会出错或崩溃。

初始化流程

从 main.m 主函数运行到 AppDelegate application，可以看到主要是 init UIWindow 初始化 bridge，然后初始化 rootView 添加到 window 里面并且调用 window 的 show 方法，其中最重要的就是初始化 bridge，调用的是 RTCBridge 的 initWithDelegate 方法

调用 initWithDelegate 判断实例之后调用 setUp 函数

setUp 函数主要是初始化 batchedBridge ，bridgeClass 是上面已经赋值的RTCxxBridge，然后调用 batchedBridge 的 start 函数

start 函数比较长，也是最主要的功能所在，可以看到，第一步是初始化了一个新的线程，传入了当前类和一个 runloop 函数，然后开启了这个线程

第二步是初始化 NativeModules ，调用 RCTGetModuleClasses 去获取当前所有的 NativeModule

以下是 RCTGetModuleClasses 的内容，可以看到是直接 copy 的 RCTModuleClasses，而下面的 RCTRegisterModule 函数是用来给 RCTModuleClasses 添加项目的，继续搜索一下 RCTRegisterModule 这个方法

可以发现在 RCT_EXPORT_MPDULE 这个宏里面，在 load 的时候执行了这个函数，而 Native 所有要暴露给 js 的方法都需要添加这个宏，所以在 NativeModule 初始化的时候就把所有的需要提供出来的 Module 都添加到了 RCTModuleClasses 里面了

后面就是调用了三个方法 ensureOnJavaScriptThread ，loadSource，executeSourceCode，在 js 线程中异步加载 js 源代码并且执行 js 代码

执行 js 是通过 c++ 的库执行的，执行的过程中会把之前所有的 RCTModuleClasses 通过 NativeModules 字段注入到 js 执行环境中作为一个全局变量，我们就可以直接在 js 里面调用 NativeModules 暴露给 js 的方法

以上就是 iOS 初始化的过程
